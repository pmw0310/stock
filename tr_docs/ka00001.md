# 국내주식 > 계좌 > 계좌번호조회 (ka00001)

현재 사용 중인 토큰을 기반으로 사용자의 계좌번호 목록을 조회하는 API입니다.[cite: 1]

## 1. 기본 정보

| 항목               | 상세 내용                                                             |
| :----------------- | :-------------------------------------------------------------------- |
| **Method**         | POST[cite: 1]                                                         |
| **URL (운영)**     | `https://api.kiwoom.com/api/dostk/acnt`[cite: 1]                      |
| **URL (모의투자)** | `https://mockapi.kiwoom.com/api/dostk/acnt` (KRX만 지원가능)[cite: 1] |
| **Format**         | JSON[cite: 1]                                                         |
| **Content-Type**   | `application/json;charset=UTF-8`[cite: 1]                             |

---

## 2. 요청 (Request) 명세

### 요청 헤더 (Header)

| Element         | 한글명       | Type   | Required | Length | Description                                                                                      |
| :-------------- | :----------- | :----- | :------: | :----: | :----------------------------------------------------------------------------------------------- |
| `api-id`        | TR명         | String |    Y     |   10   | 7자리 TR코드 (`ka00001` 입력)[cite: 1]                                                           |
| `authorization` | 접근토큰     | String |    Y     |  1000  | 토큰 타입("Bearer ")을 붙여서 호출 (예: `Bearer Egicyx...`)[cite: 1]                             |
| `cont-yn`       | 연속조회여부 | String |    N     |   1    | 응답 Header의 연속조회여부값이 Y일 경우 다음데이터 요청시 응답 Header의 cont-yn값 세팅[cite: 1]  |
| `next-key`      | 연속조회키   | String |    N     |   50   | 응답 Header의 연속조회여부값이 Y일 경우 다음데이터 요청시 응답 Header의 next-key값 세팅[cite: 1] |

### 요청 바디 (Body)

- 본 TR은 별도의 요청 바디(Input 파라미터)를 요구하지 않습니다.[cite: 1]

---

## 3. 응답 (Response) 명세

### 응답 헤더 (Header)

| Element    | 한글명       | Type   | Required | Length | Description                                   |
| :--------- | :----------- | :----- | :------: | :----: | :-------------------------------------------- |
| `api-id`   | TR명         | String |    Y     |   10   | 7자리 TR코드 (`ka00001`)[cite: 1]             |
| `cont-yn`  | 연속조회여부 | String |    N     |   1    | 다음 데이터가 있을 시 Y값 전달[cite: 1]       |
| `next-key` | 연속조회키   | String |    N     |   50   | 다음 데이터가 있을 시 다음 키값 전달[cite: 1] |

### 응답 바디 (Body)

| Element       | 한글명     | Type   | Required | Length | Description                                                                             |
| :------------ | :--------- | :----- | :------: | :----: | :-------------------------------------------------------------------------------------- |
| `acctNo`      | 계좌번호   | String |    N     |   20   | 10자리 숫자가 출력됩니다. 뒤의 2자리는 당사에서 계좌를 분류하기 위한 값입니다.[cite: 1] |
| `return_code` | 결과코드   | Number |    Y     |   -    | 거래 처리 결과 코드 (0: 정상)[cite: 1]                                                  |
| `return_msg`  | 결과메시지 | String |    Y     |   -    | 거래 처리 결과 메시지[cite: 1]                                                          |

---

## 4. 응답 예시 (Response Example)

````json
{
  "acctNo": "0123456789",
  "return_code": 0,
  "return_msg": "정상적으로 처리되었습니다"
}
```[cite: 1]

---

## 5. 유의사항
* `acctNo` 응답은 기본적으로 10자리 숫자로 구성되어 있으며, 마지막 2자리는 키움증권 내부 분류 용도로 사용됩니다.[cite: 1]
````
