# [실시간시세] 주식체결 (0B) 명세서

## 1. 기본 정보

- **API ID**: 0B
- **API 명**: 주식체결
- **대분류**: 국내주식
- **중분류**: 실시간시세
- **프로토콜**: 웹소켓 (WebSocket)
- **운영 도메인**: wss://api.kiwoom.com:10000
- **모의투자 도메인**: wss://mockapi.kiwoom.com:10000 (KRX만 지원가능)
- **URL**: /api/dostk/websocket
- **Format**: JSON
- **Content-Type**: application/json;charset=UTF-8

## 2. 개요

- 종목코드(item)를 등록하여 해당 주식의 실시간 체결 데이터(현재가, 체결량, 거래량, 호가 등)를 실시간으로 수신하는 서비스입니다.

## 3. 요청 (Request) 명세

실시간 웹소켓 연결 후, 주식체결(0B) 항목을 등록(REG)하거나 해제(REMOVE)할 때 송신하는 패킷 구조입니다.

### Request Header

| 구분   | Element       | 한글명       | Type   | Required | Length | Description                                                                               |
| :----- | :------------ | :----------- | :----- | :------- | :----- | :---------------------------------------------------------------------------------------- |
| Header | api-id        | TR명         | String | Y        | 10     | 7자리 TR코드 (ex: ka00001)                                                                |
| Header | authorization | 접근토큰     | String | Y        | 1000   | 토큰 지정 시 토큰타입("Bearer")을 붙여서 호출 (예: Bearer Egicyx...)                      |
| Header | cont-yn       | 연속조회여부 | String | N        | 1      | 응답 Header의 연속조회여부값이 Y일 경우 다음 데이터 요청 시 응답 Header의 cont-yn값 세팅  |
| Header | next-key      | 연속조회키   | String | N        | 50     | 응답 Header의 연속조회여부값이 Y일 경우 다음 데이터 요청 시 응답 Header의 next-key값 세팅 |

### Request Body

| 구분 | Element | 한글명             | Type   | Required | Length | Description                                                                                                             |
| :--- | :------ | :----------------- | :----- | :------- | :----- | :---------------------------------------------------------------------------------------------------------------------- |
| Body | trnm    | 서비스명           | String | Y        | 10     | REG: 등록, REMOVE: 해지                                                                                                 |
| Body | grp_no  | 그룹번호           | String | Y        | 4      | 사용자 정의 그룹 번호                                                                                                   |
| Body | refresh | 기존등록유지여부   | String | Y        | 1      | 등록(REG) 시 <br>0: 기존 유지 안 함 (기존 등록한 item/type 해지)<br>1: 기존 유지 (Default)<br>해지(REMOVE) 시 값 불필요 |
| Body | data    | 실시간 등록 리스트 | LIST   | Y        | -      | 실시간으로 등록하거나 해지할 요소들의 배열                                                                              |
| Body | └ item  | 실시간 등록 요소   | String | Y        | 104    | 수신을 원하는 국내주식 종목코드                                                                                         |
| Body | └ type  | 실시간 항목        | String | Y        | 2      | 실시간 항목 ID (**"0B"** 입력)                                                                                          |

## 4. 응답 (Response) 명세

웹소켓 서버로부터 실시간 데이터 수신 시, 또는 등록/해제 요청 성공 시 수신되는 패킷 구조입니다.

### Response Header

| 구분   | Element  | 한글명       | Type   | Required | Length | Description                          |
| :----- | :------- | :----------- | :----- | :------- | :----- | :----------------------------------- |
| Header | api-id   | TR명         | String | Y        | 10     | 7자리 TR코드 (ex: ka00001)           |
| Header | cont-yn  | 연속조회여부 | String | N        | 1      | 다음 데이터가 있을 시 Y값 전달       |
| Header | next-key | 연속조회키   | String | N        | 50     | 다음 데이터가 있을 시 다음 키값 전달 |

### Response Body (공통 구조)

| 구분 | Element     | 한글명            | Type   | Required | Length | Description                                                                                           |
| :--- | :---------- | :---------------- | :----- | :------- | :----- | :---------------------------------------------------------------------------------------------------- |
| Body | return_code | 결과코드          | String | N        | -      | 통신결과에 대한 코드 (등록, 해지 요청 시에만 값 전송, 0: 정상, 1: 오류. 실시간 데이터 수신 시 미전송) |
| Body | return_msg  | 결과메시지        | String | N        | -      | 통신결과에 대한 메시지                                                                                |
| Body | trnm        | 서비스명          | String | N        | -      | 등록/해지 요청 시 요청값 반환, 실시간 데이터 수신 시 **"REAL"** 반환                                  |
| Body | data        | 실시간 등록리스트 | LIST   | N        | -      | 실시간 데이터 스트리밍 배열                                                                           |
| Body | └ type      | 실시간 항목       | String | N        | -      | 실시간 항목 ID (**"0B"**)                                                                             |
| Body | └ name      | 실시간 항목명     | String | N        | -      | 실시간 항목명 (**"주식체결"**)                                                                        |
| Body | └ item      | 실시간 등록 요소  | String | N        | -      | 해당 데이터의 종목코드                                                                                |
| Body | └ values    | 실시간 값 리스트  | LIST   | N        | -      | 실시간 체결 지표 딕셔너리 (FID 데이터)                                                                |
